Cisco CX: Portable CDC Pipeline

Platform-Agnostic Change Data Capture | Polars, PySpark, Ibis/DuckDB
Snowflake SPCS (Cloud) & EC2/Docker (On-Prem) | 10M Device Telemetry Records

1. Business Context

Cisco CX processes device telemetry for 1,000+ customers (targeting 10,000 in 6 months) across multiple regions. Each customer's device fleet generates daily snapshots of telemetry data (software versions, utilization metrics, contract status). The CDC pipeline detects what changed between yesterday's snapshot and today's — identifying new devices, removed devices, and devices with updated telemetry.

The architecture must satisfy two deployment models:

  • Cloud (90% of customers): Process on Snowflake SPCS with managed compute, multi-tenant isolation, and direct Snowflake table writes
  • On-Prem / Air-Gapped (DOD, 10%): Same code runs in Docker containers on customer-controlled infrastructure with no cloud dependency

This benchmark validates that the same notebook code executes identically on both platforms, processing 10M device records and detecting 50K changes in under 60 seconds.

4. CDC Algorithm

1. Read previous snapshot (10M rows) from Snowflake — represents yesterday's device state
2. Read current snapshot (10.02M rows) from S3 parquet — today's telemetry arrival
3. Compute hash of compare columns per device:
     software_version | cpu_utilization | memory_utilization |
     critical_bugs_count | contract_status | ip_address
4. Full outer join on device_id (primary key)
5. Classify each device:
     hash_prev IS NULL       = INSERT  (new device onboarded)
     hash_curr IS NULL       = DELETE  (device decommissioned)
     hash_curr != hash_prev  = UPDATE  (telemetry values changed)
6. Extract inserts + updates from current snapshot
7. TRUNCATE target table, write changed rows via write_pandas()
